计算机系统概述

Created2022-01-05|Updated2022-01-06|408
|Post View:

基本概念

  • 计算机硬件基本组成 —— 冯 · 诺依曼模型

    1. 主机部分
      运算器、存储器、控制器
    2. 外设部分
      输入设备、输出设备

  • 操作系统
    控制和管理整个计算机系统的软硬件资源，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合
    计算机系统中最基本的系统软件

  • 操作系统的特征

    1. 并发(concurrence)
      具有处理和调度多个程序同时执行的能力
    2. 共享(sharing)
      资源共享，系统中的资源可供内存中多个并发执行的进程共同使用
      方式：互斥共享、同时访问 (宏观)
    3. 虚拟(virtual)
      虚拟技术：时分复用 (如虚拟处理器)、空分复用 (如虚拟存储器)
    4. 异步性(asynchronism)
      多道程序环境中，由于资源等因素的限制，进程是以“走走停停”的方式运行的

    并发和共享是操作系统最基本特征，两者互为存在的条件

  • 引入操作系统的目的

    1. 提供一个计算机用户与计算机硬件系统之间的接口，使计算机系统更易于使用
    2. 有效地控制和管理计算机系统中的各种硬件和软件资源，使之得到更有效的利用
    3. 合理地组织计算机系统的工作流程，以改善系统性能

  • 操作系统的基本功能

    1. 作为计算机系统资源的管理者
    • 处理机管理 (进程管理)
      进程控制、进程同步、进程通信、死锁处理、处理机调度
    • 存储器管理
      内存分配与回收、地址映射、内存保护与共享、内存扩充
    • 文件管理
      文件存储空间的管理、目录管理、文件读写管理、文件保护
      (计算机中的信息都是以文件的形式存在)
    • 设备管理 (最庞杂琐碎)
      缓冲管理、设备分配、设备处理、虚拟设备
    1. 作为用户与计算机硬件系统之间的接口
    • 命令接口
      提供一组命令供用户直接或间接控制自己的作业
      按作业控制方式分为：联机命令接口/交互式命令接口、脱机命令接口/批处理命令接口
      如：命令解释器 (shell)
    • 程序接口
      由一组系统调用 (又称广义指令) 组成
      操作系统为应用程序使用内核功能所提供的接口

    在高级语言中常以函数形式 (库函数) 提供给程序员
    但库函数属于用户程序而非系统调用，是系统调用的上层
    并非所有库函数都与系统调用有关

    图形接口也不属于操作系统，它只是为了调用系统调用而实现的功能

    1. 用作扩充机器
      裸机覆盖了软件便成为扩充机器或虚拟机

操作系统的发展与分类

手工操作阶段 (无操作系统)

  • 存在问题
    1. 手工操作的慢速与 CPU 运算的高速产生矛盾 (人机矛盾)
    2. CPU 与 I/O 设备之间速度不匹配的矛盾 (资源利用率低)

批处理阶段 (操作系统开始出现)

单道批处理系统

尽量使系统连续运行

  • 特点：自动性、顺序性、单道性
  • 优点：大大提高资源的利用率，解决人机矛盾
  • 缺点：I/O 和 CPU 计算串行，导致 CPU 利用率低下；无交互性

多道批处理系统

  • 多道程序设计技术
    特点：多道、宏观上并行、微观上串行
    需要解决：处理器分配、内存分配、I/O 设备分配、数据组织、作业组织
  • 多道批处理系统
    在单道批处理操作系统中引入多道程序设计技术
    系统资源为多个作业共享，作业之间自动调度运行，实现作业自动控制
    特点：脱机操作、成批处理 (监督程序负责自动调度)、多道程序运行 (多作业调入内存并组织其运行)
    优点：大大提高了系统资源利用率和作业吞吐量
    缺点：无交互性、付出额外开销来组织作业和切换作业
  • 批处理作业必须提供相应的作业控制信息
  • 多重处理：使用多个 CPU，可完成真正意义上的并行
    多任务处理：多个进程并发执行，宏观并行，单核多核都可
  • 批处理的主要缺点是缺少交互性

分时操作系统

  • 分时技术
    把处理器的运行时间分成很短的时间片，按时间片轮流把处理器分配给各联机作业使用，给每个用户“独占一台计算机”的感觉
  • 分时系统
    实现人机交互的系统
    目标：快速响应用户
    特点：同时性/多路性 (一台计算机与若干台终端相连)、交互性、及时性 (较短时间内响应)、“独占性”(及时性造成的表面效果)

    采用优先级 + 非抢占式调度算法

  • Unix 和 Linux 是典型的分时操作系统
  • 分时系统也是支持多道程序设计的系统

实时操作系统

  • “实时”：计算机对于外来信息能够以足够快的速度进行处理，并在被控制对象允许的时间范围内做出快速反应。对响应时间的要求比分操作系统更高
  • 特点：提供及时响应、高可靠性
  • 硬实时系统、软实时系统 (接受偶尔违反时间规定且不会引起任何永久性的损害)
  • 实时系统的进程调度，通常采用抢占式的优先级高者优先算法

以上是操作系统的基本类型
若一个操作系统兼有两种以上的功能，则该操作系统统称为通用操作系统

网络操作系统

  • 通过通信设施将物理上分散的、具有自治功能的多个计算机互连起来，实现信息交换、资源共享、可互操作和协作处理的系统
  • 主要特点：网络中各种资源共享以及各计算机之间的通信

分布式操作系统

  • 网络操作系统的更高级形式，它保持网络操作系统所拥有的全部功能，同时又具有透明性、可靠性、高性能等
  • 主要特点：分布性、并行性
  • 与网络操作系统的本质不同：
    分布式操作系统中的若干计算机相互协同完成同一任务，以缩短单个任务的执行时间来提升效率

其他操作系统

  • 个人计算机系统
    目前使用最广泛
    广泛应用于文字处理、电子表格、游戏中
    如：Windows、Linux、Macintosh
  • 嵌入式操作系统
  • 集群系统(Clustered Systerm)
    将多个独立系统耦合起来，共同完成一项任务
    通过提高单位时间内执行的任务数来提升效率

操作系统运行环境

运行机制

  • 特权指令
    只能由操作系统内核部分使用，不允许用户直接使用的指令

    1. 有关 I/O 设备使用的指令
      I/O 指令涉及中断操作，而中断处理由内核负责
    2. 有关访问程序状态的指令
      如进程切换
    3. 存取特殊寄存器指令
      涉及中断寄存器、时钟寄存器等 (置时钟指令是，读时钟指令不是)
    4. 其他特权指令
      如：清内存指令、存储保护指令、中断返回指令

  • 处理器的执行状态
    区分执行态的主要目的是保护系统程序

    1. 核心态/管态/系统态
      具有较高特权，能执行包括特权指令的一切指令，能访问所有寄存器和存储区
    2. 用户态/目态
      具有较低特权，只能执行规定的指令，只能访问指定的寄存器和存储区

  • 注意

    1. 陷入指令/访管指令是唯一只能在用户态下执行而不能在核心态下执行的指令
    2. 用户态到核心态的切换由硬件完成
    3. 用户态 → 核心态 发生在中断产生时
    4. 核心态 → 用户态 发生在中断返回用户程序时
    5. 模式切换有开销

  • 操作系统内核构成

    1. 与硬件关联较紧密的模块
      如：时钟管理、中断处理、设备驱动
    2. 运行频率较高的程序
      如：进程管理、存储器管理、设备管理

    以上是特点，不是划入内核的原因，比如进程切换主要是因为需要修改程序状态字

  • 内核 (指令操作) 的主要内容

    1. 时钟管理
      提供标准的系统时间、实现进程切换 (如时间片轮转调度)
    2. 中断机制
      只有一小部分属于内核，负责保护和恢复中断现场的信息、转移控制权到相关处理程序，这样可减少中断的处理时间，提高系统的并行处理能力
      中断处理程序只能是内核
    3. 原语(primitive)
      一些可被调用的底层公用小程序，各自完成一个规定的操作
    • 处于操作系统最底层，最接近硬件的部分
    • 程序运行具有原子性，是不被分割、不被中断执行的操作序列通常可由硬件实现
    • 运行时间较短，调用频繁
    • 通常可由硬件实现，在单处理上也可由软件通过屏蔽中断方法实现
    1. 系统控制的数据结构及处理
      系统中用来登记状态信息的数据结构：作业控制块、进程控制块、设备控制块、各类链表、消息队列、缓冲器、空闲登记区、内存分配表
      对这些数据结构的一系列操作：
    • 进程管理 (进程状态管理、进程调度和分派、创建与撤销进程控制块等)
    • 存储器管理 (存储器的空间分配和回收、内存信息保护程序、代码对换程序等)
    • 设备管理 (缓冲区管理、设备分配和回收)

  • I/O 通道
    实际上是一种特殊的处理器，具有执行 I/O 指令的能力，并通过执行通道程序来控制 I/O 操作

中断与异常

异常(exception)/内中断

处理器执行一条指令时，由处理器在其内部检测到的、与正在执行的指令相关的同步事件

  • 按原因分类

    1. 硬故障中断：由硬连线出现异常引起
      如电源掉电、存储器线路错等
    2. 软中断/程序性异常：在 CPU 内部因执行指令而引起的异常事件
      如整除 0、溢出、断点、单步跟踪、非法指令、地址越界、缺页、分时系统中的时间片中断及用户态到核心态的切换等

    浮点数运算下溢，当作机器零处理，不需要中断处理

  • 按发生异常的报告方式和返回方式不同分类

    1. 故障(Fault)：由正在执行的指令产生的异常事件 (指令启动后、执行结束前被检测到)
    • 如：指令译码时出现非法操作码、取数据时发生缺段或缺页、执行整数除法指令时发现除数为 0
    • 有的异常处理后可回到发生故障的指令继续执行 (如缺段、缺页)；有的必须终止进程的执行，因为无法通过异常处理程序恢复 (如非法操作码、除数为 0)
    1. 自陷(Trap)/陷阱/陷入
    • 人为设定的“异常”事件，在程序中事先设定一条特殊指令或通过某种方式设定特殊控制标志来人为设置一个“陷阱”，当执行到“陷阱”后，CPU 自动中止正在执行的原程序，转到相应的内核程序去执行，执行完后，再回到“陷阱”的下一条指令开始执行（当“陷阱”是转移指令时，是返回到转移目标指令执行）
    • 如：提供系统调用、程序调试功能 (设置断点、单步跟踪)
    • 这条特殊的指令称为“访管指令”(访问管理程序) 或“自陷指令”(自动调入陷阱)
    1. 终止(Abort)
    • 在执行指令过程中发生了严重错误，使程序无法继续执行，只能终止发生问题的进程，在有些严重的情况下甚至要重启系统
    • 如：控制器出现问题、访问 DRAM 或 SRAM 时发生校验错、电源掉电、线路故障等
    • 随机发生，无法确定发生异常的是哪条指令

  • 异常与 CPU 同步，基本上发生在用户态 (核心态唯一发生的异常是“缺页异常”)

  • 异常的检测是由 CPU 自身完成，不必通过外部信号通知 CPU

  • 异常不能被屏蔽，一旦出现立即处理

  • 在 IA-32 中，陷阱指令引起的异常称为编程异常
    这些指令包括 INT n、INT 3、into (溢出检查)、bound (地址越界检查) 等
    通常将 INT n 称为软中断指令，执行该指令引起的“异常”通常也称为软中断

  • 访管中断
    使用访管指令产生的自愿中断
    用户程序在用户态下要使用特权指令引起的中断属于访管中断

  • 系统调用(systerm call)
    操作系统提供给用户程序调用的接口，用户程序通过接口来获得操作系统内核提供的服务

    1. 内核提供了一系列具备预定功能的内核函数，通过系统调用呈现给用户
    2. 按功能大致分为：设备管理、进程管理、进程通信、内存管理、文件管理
    3. 执行系统调用过程
      传递系统调用参数 → 执行 trap 指令 → 执行相应服务程序 → 返回用户态
    4. 系统调用指令属于 trap 指令

    和异常一样，系统调用发生在用户态，而执行/处理在核心态

中断(Interruption)/外中断

来自 CPU (和主存) 外部、与当前 CPU 正在执行的指令无关的异步事件

  • 包括 I/O 设备发出的 I/O 中断 (如键盘输入、打印机缺纸等)、外部信号中断 (如用户按 Esc 键)，以及各种定时器引起的时钟中断

  • 外中断与内部异常两个重要的不同点

    1. 异常事件是由特定指令在执行过程中产生的；
      中断不和任何指令相关联，也不阻止任何指令的完成
    2. 异常的检测由 CPU 自身完成，不必通过外部某个信号通知 CPU；
      对于中断，CPU 必须通过总线获取中断源的标识信息，才能获知哪个设备发生了何种中断

    中断和异常本质上是一样的

  • 外中断的处理过程 (大多数)

    1. 关中断
      接下来要保护断点和现场，这期间 CPU 不应再响应其他中断请求，否则中断服务程序中断后就不能正确地恢复并继续执行现行程序
      显然，关中断是特权指令
    2. 保存断点
      唐书说断点是 PC，袁书说断点是 PC + PSW
      断点可压入堆栈，也可存入主存的特定单元中 (即由硬件实现)
    3. 引出中断服务程序
      通常有两种方法寻址中断服务程序的入口地址：
    • 硬件向量法：通过硬件产生中断向量地址 (实际上是中断类型号，而中断类型号指出了中断向量存放的地址)，再由中断向量地址找到中断服务程序的入口地址
    • 软件查询法：用软件编程的方法
    1. 保存现场和中断屏蔽字
    • 现场：用户可见的工作寄存器内容，存放着程序执行到断点处的现行值
    • 现场信息通常在中断服务程序中通过指令保存在栈中（即由软件实现）
    1. 开中断
      允许更高级中断请求得到响应，实现中断嵌套
    2. 执行中断服务程序
      中断请求的目的
    3. 关中断
      保证在恢复现场和屏蔽字时不被中断
    4. 恢复现场和屏蔽字
    5. 开中断和中断返回
      中断服务程序的最后一条指令通常是中断返回指令 (≠ 无条件转移指令)

    1~3 步由硬件自动完成 (中断隐指令)，主要目的是为了保证系统运行可靠、正确，提高处理速度也是一个好处，但不是主要目的
    4~9 步由中断服务程序完成

  • 时钟中断

    1. 主要工作
      处理和时间有关的信息及决定是否执行调度程序
    2. 和时间有关的信息
      系统时间、进程的时间片、延时、使用 CPU 的时间、各种定时器
    3. 具体来说，时钟中断服务程序更新了：
    • 内核中时钟变量的值
    • 当前进程占用 CPU 的时间
    • 当前进程在时间片内的剩余执行时间

  • 程序接口/应用编程接口 API (Application Programming Interface)
    应用程序不直接使用内核提供的系统调用，而是使用 API
    API 和系统调用没有对应关系：API 可以由 1 个或多个系统调用实现，也可以完全不用
    API 可以在各种不同的操作系统上实现，给应用程序提供完全相同的接口，但本身在这些系统上的实现却可能迥异

操作系统体系结构

大内核

  • 将操作系统的主要功能模块都作为一个联系紧密的整体运行在核心态，从而为应用提供高性能的系统服务
  • 优点
    1. 系统的效率相对较高
      因各管理模块之间共享信息，能有效利用相互之间的有效特性，进而为应用提供高性能的系统服务
    2. 接口简单直接
  • 缺点 —— 面临“软件危机”困境
    1. 高耦合导致可移植性差
    2. 复杂性随着系统规模增大而迅速增长，可适应性较差，难维护

微内核

  • 主要思想：内核只保留最基本功能 (如进程管理等)，将其他服务尽可能从内核中分离出去，用若干个运行在用户态下的进程 (即服务器进程) 来实现，形成所谓的 C/S 模式。这些服务器进程的执行相互独立，交互则借助微内核进行通信
  • 优点：可靠；灵活；便于维护；适合分布式处理的计算环境
  • 缺点：性能问题，效率不高 (进程通信，核心态和用户态之间频繁切换造成的开销)
